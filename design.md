# Scholarship Assistant – System Design

## Architecture
The system uses a simple client-server architecture.

## Components
- Frontend: HTML, CSS, JavaScript
- Backend: Python (FastAPI)
- Logic: Rule-based eligibility check

## Workflow
1. User enters details
2. Data is sent to backend
3. Rules are applied
4. Result is returned

## Security
- Input validation
- No permanent data storage

## Scalability
- Can add more scholarships easily
